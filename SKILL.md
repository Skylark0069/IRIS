# IRIS Skincare Compounds Search

Search the IRIS (Integrated Risk Intelligence System for Skincare Compounds) database via a REST API running on the campus network. IRIS is a proactive governance platform for aquatic toxicity risks in skincare chemicals, built on a global database of 11,110 skincare-related compounds, 21,035,379 associated patents, and 94,658 predicted environmental transformation products.

This skill handles querying only — the backend server is already deployed.

## Server

- **Base URL**: `http://114.212.79.90:8000`
- All endpoints return JSON with `{"ok": true, ...}` on success.
- HTTP 400 means invalid parameters; HTTP 404 means not found.

## Key Terminology

Understanding these terms is essential for interpreting API results correctly.

### HPAT — Highly Potent Aquatic Toxicant

A parent skincare compound whose minimum LC₅₀ value across three trophic levels (fish, algae, and Daphnia/Water Flea) is less than 1 mg/L. This threshold identifies compounds with high acute aquatic toxicity. HPAT status can be `True`, `False`, or `Unknown` (when insufficient experimental or predicted data exists).

Toxicity data comes from two sources:
- **Experimental**: Retrieved from the ECOTOX database (mortality/MOR endpoint, LC₅₀)
- **Predicted**: Computed via the ECOSAR model when experimental data is unavailable

### TP-HPAT — Transformation Product-Derived Highly Potent Aquatic Toxicant

A parent skincare compound for which at least one of its predicted environmental or biological transformation products exhibits high aquatic toxicity (LC₅₀ < 1 mg/L). Transformation products are predicted using BIOTRANSFORMER 3.0 across multiple modes (EC-based, CYP450, Phase II, human gut, all-human, and environmental). Unlike HPAT, which measures the parent compound's own toxicity, TP-HPAT captures latent risk — a compound may itself be benign but degrade into highly toxic byproducts after environmental release.

### SPCN — Skincare Patent Co-occurrence Network

A graph-based framework where each node represents an individual skincare compound and each edge is weighted by the number of patents in which two compounds co-occur. The SPCN couples a compound's inherent aquatic toxicity with its topological importance in industrial innovation — a proxy for future market prevalence and environmental exposure potential. The network covers annual snapshots across five jurisdictions: China (CHN), European Union (EUR), Japan (JPN), Republic of Korea (KOR), and United States (USA).

### KEY NODE

A compound node whose PageRank value ranks in the top 10% within the SPCN over a future prediction window (typically the T+3 period, i.e., the three years following a given reference year). KEY NODEs represent compounds predicted to become disproportionately important in future skincare formulation innovation.

### PageRank

The core topological metric used to quantify compound importance within the SPCN. Node-removal experiments across all tested jurisdictions and time periods confirmed that PageRank-based prioritization achieves the optimal reduction in network-level aquatic toxicity influence, outperforming weighted degree, clustering coefficient, degree centrality, eigenvector centrality, betweenness centrality, and Katz centrality.

### KEY LIST

A priority inventory of compounds requiring targeted regulatory or R&D intervention. The KEY LIST is generated by:
1. Using the FKP-TabPFN model to predict future KEY NODEs from historical SPCN data (T-5 year input window predicting T+3 output window)
2. Filtering the predicted KEY NODEs to retain only those that are HPATs or TP-HPATs

Validation results:
- Achieved up to **75% coverage** of HPATs and **80% coverage** of TP-HPATs in all registered US skincare products (2013–2018)
- Achieved **100% coverage** for high-prevalence HPATs and TP-HPATs (compounds in the top 10% by formulation occurrence frequency)
- Exceeded **85% coverage** (maximum 95%) for HPATs in skincare products imported into China (2024–2025)

### Additional KEY LIST

Compounds newly appearing in a region's KEY LIST that have never previously been identified as KEY NODEs in that jurisdiction. These represent emerging risks from novel innovation trajectories. Among Additional KEY LIST compounds for the 2025–2027 prediction window, 16 compounds carry GHS aquatic hazard statements (H400, H410, H411, H412, H413).

### FKP-TabPFN Model

The Future Key-PageRank TabPFN model — a transformer-based Prior-Data Fitted Network pre-trained on synthetic tabular tasks. It predicts future KEY NODEs by integrating multiple temporal trend features from historical SPCN topologies (Mann-Kendall statistics, Pearson/Spearman correlations, Sen's slope, etc. computed over T-5 year windows). Achieves **96.27% classification accuracy** on held-out test data and consistently outperforms Logistic Regression, KNN, Naive Bayes, Decision Tree, Random Forest, Gradient Boosting, XGBoost, LightGBM, and MLP baselines.

### GHS Aquatic Hazard Statements

Globally Harmonized System hazard codes relevant to aquatic toxicity found among predicted KEY LIST compounds:
- **H400**: Very toxic to aquatic life
- **H410**: Very toxic to aquatic life with long lasting effects
- **H411**: Toxic to aquatic life with long lasting effects
- **H412**: Harmful to aquatic life with long lasting effects
- **H413**: May cause long lasting harmful effects to aquatic life

## How to Query

Use `mcp__workspace__web_fetch` to call the API. Always parse the JSON response.

```
mcp__workspace__web_fetch with url = "http://114.212.79.90:8000/api/endpoint"
```

## API Reference

### 1. Compound Search

**GET** `/api/compound?q={query}&field={field}`

Search compounds by a specific field or across all fields.

| Param | Required | Description |
|-------|----------|-------------|
| `q` | yes | Search value (ID, name, CAS, SMILES, CID) |
| `field` | no | One of: `ID`, `SMILES`, `CID`, `CAS`, `NAME`. Omit to search all fields. |

Examples:
- `/api/compound?q=100&field=ID`
- `/api/compound?q=aspirin&field=NAME`
- `/api/compound?q=50-00-0&field=CAS`
- `/api/compound?q=glycerol` (auto-search across all fields)

Response: `{"ok": true, "query": "...", "query_type": "...", "count": N, "results": {"compound_id": {...}}}`

### 2. Get Compound by ID

**GET** `/api/compound/{compound_id}`

Returns a single compound. Equivalent to `/api/compound?q={id}&field=ID`.

### 3. List Compounds by Criteria

**GET** `/api/compounds/{list_type}`

Retrieve pre-filtered lists of compounds. These endpoints correspond to the KEY LIST predictions and toxicity classifications generated by the IRIS platform.

| list_type | Meaning |
|-----------|---------|
| `hpat` | All HPAT = True compounds (Highly Potent Aquatic Toxicants; LC₅₀ < 1 mg/L across fish, algae, Daphnia) |
| `tp_hpat` | All TP-HPAT = True compounds (parent compounds with at least one highly toxic transformation product) |
| `cn_key` | KEY LIST for China — predicted future KEY NODEs that are also HPAT/TP-HPAT (future_key_AREA contains CN) |
| `eu_key` | KEY LIST for the European Union (future_key_AREA contains EUR) |
| `jp_key` | KEY LIST for Japan (future_key_AREA contains JPN) |
| `kr_key` | KEY LIST for Korea (future_key_AREA contains KOR) |
| `us_key` | KEY LIST for the United States (future_key_AREA contains USA) |

Response: `{"ok": true, "type": "...", "count": N, "compounds": [{"_id": "...", "SMILES": "...", "HPAT": "...", "TP-HPAT": "..."}]}`

### 4. Patent Search

**GET** `/api/patents/search?q={keyword}`

Fuzzy search across patent titles, abstracts, and matched function names/evidence. Results sorted with HPAT=True first.

Example: `/api/patents/search?q=anti-aging`

Response: `{"ok": true, "keyword": "...", "count": N, "compounds": [{"_id": "...", "SMILES": "...", "HPAT": "...", "TP-HPAT": "..."}]}`

### 5. List All Functions

**GET** `/api/functions`

Returns all unique, normalized function names across all patents (deduplicated).

Response: `{"ok": true, "count": N, "functions": ["Acne Treatment", "Anti Aging", ...]}`

### 6. Search by Function

**GET** `/api/functions/{function_name}`

Find compounds whose patents include a specific function. The function name is normalized for matching (case-insensitive, ignores leading numbers, normalizes separators). Results sorted with HPAT=True first.

Example: `/api/functions/moisturizing`

Response: `{"ok": true, "function_name": "Moisturizing", "count": N, "compounds": [...]}`

## Response Fields

Compound result objects include:
- `_id`: Compound ID
- `SMILES`: Chemical structure (Simplified Molecular-Input Line-Entry System)
- `HPAT`: Highly Potent Aquatic Toxicant status (True/False/Unknown) — see Terminology section for full definition
- `TP-HPAT`: Transformation Product-Derived HPAT status (True/False/Unknown) — see Terminology section for full definition
- `cid`: PubChem CID (full results only)
- `CAS`: CAS Registry Number (full results only)
- `name`: Compound name (full results only)
- `patents`: Patent data including co-occurrence information (full results only)
- `future_key_AREA`: Target markets where this compound appears in the predicted KEY LIST (full results only). Values are region codes: CN, EUR, JPN, KOR, USA.

## Display Guidelines

When presenting results:
1. State the total count first
2. For compound lists, show a summary table: ID, name/SMILES, HPAT, TP-HPAT
3. When HPAT or TP-HPAT is True, highlight this prominently — these are high-priority compounds from an aquatic toxicity perspective
4. For KEY LIST results (`cn_key`, `eu_key`, `jp_key`, `kr_key`, `us_key`), explicitly mention that these are predicted future high-priority compounds combining network importance with aquatic toxicity
5. For function lists, show all function names grouped or as a readable list
6. For patent searches, mention the keyword and count
7. When no results, say so clearly and suggest trying broader terms
8. HPAT=True compounds are always sorted first in patent/function searches
9. When a compound has TP-HPAT=True but HPAT=False, note that while the parent compound itself is not highly toxic, its environmental transformation products are — this represents latent aquatic risk

## Background: The IRIS Platform

IRIS is a proactive governance framework that shifts aquatic toxicity risk assessment for skincare chemicals from reactive post-marketing control to pre-innovation intervention. Instead of waiting for environmental damage to be detected after commercial release, IRIS uses patent co-occurrence network analysis and machine learning to forecast which compounds will become both widely adopted and environmentally hazardous — before they enter mass production.

The system operates in four layers:
1. **DATA LAYER**: A multidimensional database of 11,110 skincare compounds with patent data, biotransformation products (94,658 TPs predicted via BIOTRANSFORMER), aquatic biotoxicity profiles (ECOTOX experimental + ECOSAR predicted), LC-MS/MS spectra (GNPS/MassBank/MONA experimental + CFM-ID predicted), and regulatory prohibition status across China, EU, Korea, and the US.
2. **NETWORK LAYER**: The Skincare Patent Co-occurrence Network (SPCN) where PageRank quantifies each compound's topological importance — a proxy for future market prevalence.
3. **MODEL LAYER**: The FKP-TabPFN model (96.27% accuracy) predicts future KEY NODEs from historical network dynamics (T-5 → T+3 prediction window).
4. **PREDICTION LAYER**: KEY LIST generation — filtering predicted KEY NODEs to retain only HPATs and TP-HPATs, producing a prioritized intervention inventory.

Regional prohibited ingredient lists currently show low coverage of identified HPATs and TP-HPATs, as existing regulations are not primarily oriented toward aquatic ecotoxicity. The IRIS database bridges this gap, and the KEY LIST provides a forward-looking, aquatic-toxicity-oriented supplementary framework for regulatory assessment.

1. Understand what the user wants: a specific compound, a list by criteria (HPAT, TP-HPAT, KEY LIST), patent search, or function lookup
2. Choose the matching endpoint from the reference above
3. Build the URL and call `mcp__workspace__web_fetch`
4. Parse the JSON, extract key fields
5. Present results clearly, highlighting HPAT/TP-HPAT status and KEY LIST membership

7. If the user wants more detail on a compound, follow up with `/api/compound/{id}`
